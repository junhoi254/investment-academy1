내 컴퓨터가 아닌 **인터넷 서버**에 투자학당을 올려서

### 결과물:
```
✅ https://투자학당.railway.app (자동 생성 주소)
```

## 🎬 STEP 1: GitHub 계정 만들기 (5분)

### GitHub이 뭔가요?
→ 코드를 저장하는 인터넷 저장소입니다.  
→ 무료이고, 전세계 개발자들이 사용합니다.

### 1-1. 웹사이트 접속

1. **브라우저 열기** (Chrome, Edge, Safari 등)
2. 주소창에 입력: `github.com`
3. Enter

### 1-2. 회원가입

화면 오른쪽 위에 **"Sign up"** 버튼 보임
→ 클릭!

**페이지 1: 이메일 입력**
```
Email address: 여러분의이메일@gmail.com
```
→ **"Continue"** 클릭

**페이지 2: 비밀번호 입력**
```
Create a password: ******** (8자 이상, 기억하기 쉬운 것)
```
→ **"Continue"** 클릭

**페이지 3: 사용자 이름**
```
Username: investment-teacher (또는 원하는 이름, 영문만)
```
→ **"Continue"** 클릭

**페이지 4: 이메일 수신**
```
Email preferences: n (광고 안 받기)
```
→ **"Continue"** 클릭

**페이지 5: 로봇 확인**
```
"퍼즐 풀기" 또는 "Start puzzle" 클릭
→ 간단한 퍼즐 풀기
```

**페이지 6: 가입 완료**
```
"Create account" 클릭!
```

### 1-3. 이메일 인증

1. **이메일 받은편지함 열기**
2. GitHub에서 온 메일 찾기
   ```
   제목: [GitHub] Please verify your email address
   ```
3. 메일 열기
4. **초록색 버튼 클릭**: "Verify email address"

→ 브라우저로 자동 이동!

### 1-4. 간단한 설문 (건너뛰기 가능)

```
Just me / With a team → "Just me" 선택
Student / Teacher / Other → 아무거나 선택
→ "Continue" 클릭

또는 상단 "Skip personalization" 클릭하면 건너뛰기 가능
```

### 1-5. 완료!

→ GitHub 대시보드 화면 표시됨! ✅
→ 왼쪽 위에 고양이 로고 보임

## 📤 STEP 2: GitHub Desktop 설치 (5분)

### GitHub Desktop이 뭔가요?
→ 코드를 GitHub에 쉽게 올릴 수 있는 프로그램입니다.  
→ 명령어 입력 없이 클릭만으로 가능!

### 2-1. 다운로드

1. **새 탭 열기**
2. 주소창에 입력: `desktop.github.com`
3. Enter

4. **보라색 버튼 클릭**: 
   - Windows: "Download for Windows"
   - Mac: "Download for macOS"

5. 파일 다운로드 대기 (1-2분)

### 2-2. 설치

**Windows:**
1. 다운로드된 "GitHubDesktopSetup.exe" 더블클릭
2. 자동으로 설치 시작
3. 2-3분 대기
4. 설치 완료 후 자동 실행

**Mac:**
1. 다운로드된 "GitHubDesktop.zip" 더블클릭
2. Applications 폴더로 드래그
3. Applications에서 GitHub Desktop 더블클릭
4. "열기" 클릭

### 2-3. GitHub 로그인

**"Welcome to GitHub Desktop" 화면:**

1. **"Sign in to GitHub.com"** 클릭
2. 브라우저 자동으로 열림
3. GitHub 비밀번호 입력
4. **"Authorize desktop"** 클릭
5. GitHub Desktop으로 자동 복귀

**"Configure Git" 화면:**
```
Name: 투자학당 (또는 본인 이름)
Email: 여러분의이메일@gmail.com
```
→ **"Continue"** 클릭

→ GitHub Desktop 준비 완료! ✅

## 📂 STEP 3: 투자학당 코드 GitHub에 올리기 (5분)

### 3-1. 폴더 추가

**GitHub Desktop 화면에서:**

1. 가운데 **"Add an Existing Repository"** 클릭
   (또는 File → Add Local Repository)

2. **"Choose..."** 버튼 클릭

3. **폴더 선택 창 열림:**
   - 바탕화면으로 이동
   - **"investment-academy"** 폴더 클릭
   - **"폴더 선택"** 또는 **"Select Folder"** 클릭

4. **오류 메시지 나타날 수 있음:**
   ```
   This directory does not appear to be a Git repository
   ```
   → 괜찮습니다! **"create a repository"** 클릭

5. **"Create a Repository" 창:**
   ```
   Name: investment-academy (자동 입력됨)
   Description: 투자학당 - 일타훈장님 리딩방
   □ Keep this code private (체크 해제)
   Git Ignore: None
   License: None
   ```
   → **"Create Repository"** 클릭

→ 왼쪽에 파일 목록 나타남! ✅

### 3-2. GitHub.com에 업로드

**GitHub Desktop에서:**

1. 상단 **"Publish repository"** 버튼 클릭

2. **팝업 창:**
   ```
   Name: investment-academy
   Description: 투자학당 - 일타훈장님 리딩방
   ☐ Keep this code private (체크 해제 - 공개로!)
   Organization: (비워두기)
   ```

3. **"Publish Repository"** 클릭

**업로드 시작!**
```
Uploading...
Publishing repository to GitHub...
```
→ 2-5분 소요 (파일 크기에 따라)

**완료 메시지:**
```
✅ Published investment-academy to GitHub
```

### 3-3. 확인

1. 브라우저에서 GitHub.com 열기
2. 로그인되어 있는지 확인
3. 왼쪽 메뉴 **"Repositories"** 클릭
4. **"investment-academy"** 보임!

→ GitHub 업로드 완료! ✅

## 🚂 STEP 4: Railway 가입 및 배포 (5분)

### Railway가 뭔가요?
→ 코드를 자동으로 실행시켜주는 무료 서버입니다.  
→ GitHub와 연결하면 클릭 한 번으로 배포!

### 4-1. Railway 접속

1. **새 탭 열기**
2. 주소창에 입력: `railway.app`
3. Enter

### 4-2. 가입

**Railway 메인 화면:**

1. 오른쪽 위 **"Login"** 클릭

2. **"Login with GitHub"** 클릭 (검은색 버튼)

3. GitHub 비밀번호 입력 (자동 로그인되어 있으면 생략)

4. **"Authorize Railway"** 클릭 (초록색 버튼)
   ```
   Railway wants to:
   - Read your repositories
   - Access your email
   ```

5. Railway 대시보드로 자동 이동!

→ Railway 가입 완료! ✅

## 🚀 STEP 5: 투자학당 배포하기 (실전!)

### 5-1. 새 프로젝트 만들기

**Railway 대시보드:**

1. 가운데 큰 버튼 **"New Project"** 클릭 (보라색)

2. 메뉴 나타남:
   ```
   Deploy from GitHub repo
   Deploy from Template
   Empty Project
   ```

3. **"Deploy from GitHub repo"** 클릭!

4. 저장소 목록 나타남:
   ```
   Configure GitHub App (설정 필요시)
   investment-academy (여러분의 저장소)
   ```

5. **"investment-academy"** 클릭!

**자동 배포 시작!** 🚀
```
Setting up project...
Deploying investment-academy...
```

### 5-2. 배포 진행 확인

**화면에 2개 카드 나타남:**

```
┌─────────────────┐  ┌─────────────────┐
│    backend      │  │    frontend     │
│   (Python)      │  │   (Node.js)     │
│                 │  │                 │
│   🔄 Deploying  │  │   🔄 Deploying  │
└─────────────────┘  └─────────────────┘
```

**각 카드 클릭하면 배포 로그 보임:**
```
Installing dependencies...
Building application...
Starting server...
```

→ 5-10분 소요 (커피 타임 ☕)

### 5-3. Backend 설정

**중요! 환경 변수 설정해야 합니다!**

1. **"backend"** 카드 클릭

2. 상단 탭에서 **"Variables"** 클릭

3. **"New Variable"** 버튼 클릭 (오른쪽 위)

4. **다음 변수들 하나씩 추가:**

**첫 번째 변수:**
```
VARIABLE_NAME: SECRET_KEY
VALUE: my-super-secret-key-12345
```
→ **"Add"** 클릭

**두 번째 변수:**
```
VARIABLE_NAME: DATABASE_URL
VALUE: sqlite:///./trading_chat.db
```
→ **"Add"** 클릭

**세 번째 변수:**
```
VARIABLE_NAME: MT4_API_KEY
VALUE: my-mt4-api-key-12345
```
→ **"Add"** 클릭

**네 번째 변수:**
```
VARIABLE_NAME: PORT
VALUE: 8000
```
→ **"Add"** 클릭

5. 모든 변수 추가 후 **자동으로 재배포 시작!**

### 5-4. Backend 도메인 확인

1. 상단 탭 **"Settings"** 클릭

2. 아래로 스크롤해서 **"Domains"** 섹션 찾기

3. **"Generate Domain"** 버튼 클릭

4. **URL 자동 생성됨!** (예시)
   ```
   https://backend-production-xxxx.up.railway.app
   ```

5. **이 URL 복사!** (중요! 다음 단계에서 사용)
   - URL 위에 마우스 올리면 📋 복사 아이콘 나타남
   - 클릭해서 복사

→ 메모장에 붙여넣어 두기!

### 5-5. Frontend 설정

1. 뒤로가기 (왼쪽 위 화살표) → 프로젝트 대시보드

2. **"frontend"** 카드 클릭

3. **"Variables"** 탭 클릭

4. **"New Variable"** 클릭

**첫 번째 변수:**
```
VARIABLE_NAME: REACT_APP_API_URL
VALUE: https://backend-production-xxxx.up.railway.app
```
⚠️ 위에서 복사한 backend URL 붙여넣기!
→ **"Add"** 클릭

**두 번째 변수:**
```
VARIABLE_NAME: REACT_APP_WS_URL
VALUE: wss://backend-production-xxxx.up.railway.app
```
⚠️ https를 wss로 바꾸기! (나머지는 동일)
→ **"Add"** 클릭

5. **자동으로 재배포!**

### 5-6. Frontend 도메인 생성

1. **"Settings"** 탭 클릭

2. **"Domains"** 섹션 찾기

3. **"Generate Domain"** 클릭

4. **URL 생성됨!** (예시)
   ```
   https://frontend-production-yyyy.up.railway.app
   ```

→ 이게 여러분의 투자학당 주소입니다! 🎉

## 🐛 문제 해결

### Q: Railway 배포가 실패했어요

**확인 사항:**
1. Railway 대시보드 → Backend/Frontend 카드 클릭
2. **"Deployments"** 탭 클릭
3. 최근 배포 클릭
4. 빨간색 오류 메시지 확인

**자주 발생하는 오류:**
```
Module not found
→ requirements.txt 파일 확인

Port already in use
→ PORT 환경 변수 확인 (8000)

Build failed
→ 로그 확인, GitHub에 모든 파일 업로드 확인
```

### Q: 로그인이 안 돼요

1. Backend가 정상 작동하는지 확인
2. Backend URL/api/docs 접속
3. API 문서 페이지 뜨면 정상

### Q: 채팅이 안 돼요

1. Frontend 환경 변수 확인
2. REACT_APP_WS_URL이 wss://로 시작하는지 확인
3. Backend URL과 동일한지 확인 (https를 wss로만 변경)

### Q: Railway 무료 한도가 궁금해요

**무료 플랜:**
```
✅ 월 $5 크레딧
✅ 약 500시간 실행
✅ 100GB 대역폭
✅ 동시 접속 50명 가능
```

**초과 시:**
- 이메일로 알림
- 유료 플랜 선택 ($5/월~)
- 또는 다른 무료 서비스로 이동
